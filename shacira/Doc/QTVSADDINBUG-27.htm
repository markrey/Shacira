The Visual Studio Add-in should have a button to convert a "regular" vcproj to a Qt-enabled vcproj

Description

Currently, there is no easy way to convert non-Qt vcproj files into a Qt-enabled vcproj files. By Qt-Enabled the following is meant:

On a non-Qt-enabled vcproj, when right clicking on the project name under Solution Explorer, all options related to Qt are grayed-out. It would be really useful if a way of converting .vcproj files to Qt-Enabled ones was added to the Add-in.

There is a workaroud for this issue.
use any kind of texteditors to open yourproject.vcproj files.
after line
RootNamespace="yourproject"
add a new attrib like this:
Keyword="Qt4VSv1.0"
then your project will treat as a Qt-enabled project.
you may need re-open it in visual studio to take effect.

Etienne Savard added a comment - 14/Oct/11 8:35 PM

With VS2010, the workaround is slighly different.

1) Open yourproject.vcxproj with a text editor
2) Locate the tag : <PropertyGroup Label="Globals">
3) Between <PropertyGroup Label="Globals"> and </PropertyGroup> add the following line :
<Keyword>Qt4VSv1.0</Keyword>
4) Reload your project in VS2010
5) In solution explorer, righ-click on your project
6) In the menu, select "Convert project to Qt Add-in project"

Revise your Qt Project Settings to include the modules you need and then you are ready to go.

Peter Bienert added a comment - 05/Dec/11 9:16 AM

As JÃ¶rg declared the Bug 102 as duplicate of this one, some remarks:
I used only the solution with RootNamespace in 2010, but this is for 2008. QT VS Addin does detect the project as QT Project in VS2010 ! The other solution "Globals" for 2010 has absolute same behaviour. In menu QT, you can now select "convert to vsaddin project". Only thing happens, is that this is included at the end of the vsxproj file:
<ProjectExtensions>
<VisualStudio>
<UserProperties QtVersion_x0020_x64="qt474_64" />
</VisualStudio>
// I worked with 64 bit
Nothing more ! And VSAddIn forgot to change the xxx.vcxproject.filters, without these, nothing works.
My workaraound is this:
Create a regular qt project, i.e. Gui-Project and name it "dummy". Now you get a full working .vcxproj + vcxproj.filters.
Now edit your existing project and add the <Keyword> to the propertygroup Label="globals". Load the project, and select in menu qt "convert to vsaddin project".
Next step is to close project, use editor and load the existing project filters file, and as second file load the dummy filters file. Now select the filters block
<ItemGroup>
<Filter Include="Source Dateien">
<UniqueIdentifier>{4FC737F1-C7A5-4376-A066-2A32D752A2FF}</UniqueIdentifier>
<Extensions>cpp;cxx;c;def</Extensions>
</Filter>
...
</ItemGroup> and replace the existing filters block.
Save and reload the project. Now it is annoying, because the addin does not work correctly, when you had still things in your "header files", these are not converted. You have to delete from project and than reassign to the same "header files" again, to get the mocs...
This normally should be done by the vsaddin and not by hand....
Regards,
Peter
